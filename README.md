# SeiyuuRoleProfiler

Interactive seiyuu lookup built from character-description language.

The pipeline is intentionally explainable:

1. collect popular AniList characters and Japanese voice credits;
2. clean character descriptions into POS-gated descriptor n-grams;
3. fit a global TF-IDF matrix over adjective one-word descriptors and 2- to
   6-word character-description phrases;
4. decompose that matrix with global K=96 NMF topics;
5. aggregate each seiyuu's characters over those shared topic lanes;
6. rank lanes by enrichment against the global character-topic baseline.

The checked-in compact cache and fitted K=96 model are enough to build and
serve the profiler without the large research dumps.

Descriptor n-grams are generated by a non-bespoke POS gate over standalone
one-word adjectives and 2- to 6-word noun/gerund phrases. Adjective-modified
multi-word chunks such as `black hair` are used only as suppressors: they are
not emitted as features, and their contained adjective does not leak out as a
one-word descriptor. Standalone gerunds, pronouns, finite verbs, contraction
fragments, stopwords, and proper nouns are excluded. The analyzer also uses
document-level longest-match suppression inside that 1- to 6-gram range, so a
character with `student council president` does not also receive TF-IDF weight
for `student`, `president`, `student council`, or `council president`.

The source character pool is filtered to AniList characters with at least 100
favourites, but seiyuu profiles are still portfolio-scored. Homepage samples
require at least 10 described characters and 20 roles, and displayed lanes
require at least two supporting characters for the seiyuu. This keeps
single-role outliers from ranking above real seiyuu portfolios.

## Quick Start

```sh
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python scripts/build_profiles.py
.venv/bin/python scripts/serve.py
```

Then open `http://127.0.0.1:8765/`.

## Data

- `data/role_edges.json`: compact seiyuu-character-anime credit cache.
- `models/k96_pos_adj1to6_descriptors/`: POS-gated global TF-IDF + K=96 NMF artifacts.
- `site/profiles.json`: generated seiyuu profile payload for the page.

The compact cache links anime to MAL when a MAL anime id is available. Character
and seiyuu pages currently link to AniList because this cache does not include
MAL character/person ids.

## Rebuild

Build the profile payload and static page:

```sh
.venv/bin/python scripts/build_profiles.py
```

Refit topics from raw AniList character and description dumps:

```sh
.venv/bin/python scripts/fit_topics.py \
  --characters-input path/to/characters.json \
  --descriptions-input path/to/descriptions.json \
  --topic-counts 96
```

For reproducible feature construction, materialize the descriptor features
before fitting NMF:

```sh
.venv/bin/python scripts/cache_legal_ngrams.py \
  --characters-input path/to/characters.json \
  --descriptions-input path/to/descriptions.json

.venv/bin/python scripts/cache_character_ngram_matrix.py

.venv/bin/python scripts/fit_topics.py --topic-counts 96
```

The first runner writes `legal_ngrams.json`, `legal_ngram_vocabulary_raw.json`,
`character_rows.json`, and the adjective suppression audit. The second runner
writes `character_ngram_tfidf.npz`, `tfidf_vocabulary.json`, and matrix
metadata. The NMF runner reuses that cached matrix when present.

Generate cached semantic labels for the fitted topics with a local Ollama model.
This uses only topic n-grams and weights, not exemplar characters:

```sh
ollama pull qwen3.5:4b
.venv/bin/python scripts/label_topics.py
.venv/bin/python scripts/build_profiles.py
```

Cache AniDB character-tag enrichment for the current role corpus:

```sh
.venv/bin/python scripts/cache_anidb_character_tags.py --fetch-anime-xml
```

See `ANIDB_TAG_CACHE.md` for the API/client setup, cache layout, and the
character-tag fallback caveat.

Cache legal VNDB character-trait enrichment for the current role corpus:

```sh
.venv/bin/python scripts/cache_vndb_character_traits.py
```

See `LEGAL_CHARACTER_TAG_SOURCES.md` for source tradeoffs and provenance.

Extract multi-label role/personality/trait tags from character descriptions
with a local Ollama model:

```sh
.venv/bin/python scripts/cache_llm_character_tags.py \
  --descriptions-input path/to/descriptions.json
```

See `LLM_CHARACTER_TAGS.md` for the schema, cache layout, and reproducibility
notes.

Recreate the compact credit cache from the large research dumps:

```sh
.venv/bin/python scripts/prepare_compact_cache.py \
  --credit-splits-input path/to/credit-splits.json \
  --characters-input path/to/characters.json \
  --output data/role_edges.json
```

## Deferred

When explaining why a character belongs under a lane, display the intersection
between that character's own descriptor n-grams and the lane's high-weight
n-grams. This avoids implying that a full global topic label applies when only
the topic's generic head noun matched.
